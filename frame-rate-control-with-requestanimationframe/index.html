Frame rate control and motion with requestAnimationFrame

So often I see JS game/animation loops, running off requestAnimationFrame, which update logic AND render every frame, without enforcng a consistent refresh rate. This means that your game will actually execute slower (including AI, actual speed of movement, etc) for someone with a lower refresh rate (say, someone on a low-powered device).

This Timer class enforces a set rate of refresh so that even if the frame rate is sub-optimal, the actual speed of updates (e.g. to position) remains constant. The three boxes are all moving at the same relative speed, even though the motion itself may not be as smooth for lower FPSes.
